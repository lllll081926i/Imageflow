## 一、矢量图 (SVG) 转位图 / ICO 专题审查与修复

### 1. 问题复盘
此前在进行矢量图（SVG）转位图或 ICO 格式转换时，由于以下原因容易抛出异常报错：
1. **尺寸解析边界问题**：部分 SVG 文件缺少显式 `width` / `height` 属性（仅包含 `viewBox`，或没有任何尺寸声明），导致解析尺寸时抛出 `TypeError` 或生成 `(0, 0)` 无效图像。
2. **ICO 多尺寸分配逻辑**：SVG 转 ICO 时，ICO 的 `ico_sizes` 参数（如 `[16, 32, 64, 128]`）未与 SVG 的 `_calculate_svg_render_size` 渲染画布尺寸打通，导致 ICO 引擎生成图像时尺寸失配。

### 2. 治理与修复措施
1. **智能尺寸回退**：在 `converter.py` 的 `_parse_svg_intrinsic_size_from_attrs` 中增加 `viewBox` 计算逻辑及 Default 1024x1024 兜底机制，支持包含 `100%` 相对尺寸单位的 SVG。
2. **ICO 渲染尺寸匹配**：在 `_calculate_svg_render_size` 中增加 `format_type == "ico"` 判断，根据 `ico_sizes` 中的最大边长动态调整 SVG 渲染像素基准，确保生成的 ICO 图标清晰度最高。
3. **SVG 格式转换全引擎通配**：
   - 验证了 `Converter`（格式转换）、`Adjuster`（图像调节）、`Watermark`（水印）、`Filter`（滤镜）四大引擎对 SVG 输入的完备支持。
   - 补齐了 `test_svg_to_ico_conversion_success` 单元测试至 `backend/tests/engines/test_converter.py`。

## 二、前后端核心质量与隐患治理

### 1. 前端（React + TypeScript + Vite）

#### (1) 全量开启 TypeScript 严格模式 (`strict: true`)
- **变更文件:** `frontend/tsconfig.json`
- **治理项:** 修复了 `InfoSettingsPanel.tsx`、`DetailView.tsx`、`useGifResizeState.ts` 和 `useImagePreview.ts` 中的 7 处 strict 类型推断隐患，确保对象方法可选链调用 (`?.()`) 与隐式 `any` 清零。

#### (2) 消除组件内联声明反模式 (React Performance)
- **变更文件:** `frontend/components/Controls.tsx`
- **治理项:** 移除了在 `Controls` 渲染函数内部内联声明 `const SortMenu = () => ...` 的反模式，重构为 `createPortal` 挂载，避免每次父组件重新渲染时反复销毁与重建子组件 DOM 节点。

#### (3) 用户主题偏好持久化 (Theme Persistence)
- **变更文件:** `frontend/App.tsx`
- **治理项:** 补充 `localStorage` 读写逻辑，解决用户手动切换暗色/亮色主题后，刷新或重启桌面应用时偏好设置重置为系统默认的问题。

#### (4) Object URL 异步撤销防护
- **变更文件:** `frontend/components/DetailView.tsx`
- **治理项:** 在 JSON 导出事件处理中将 `URL.revokeObjectURL(url)` 改为 `setTimeout` 延后 1 秒撤销，避免在部分 Chromium / WebKit Event Loop 调度下出现点击下载失败的边缘问题。

---

### 2. 后端（Python + pywebview + Pillow）

#### (1) 图像通道 (Image Split) 资源泄漏治理
- **变更文件:** `backend/engines/adjuster.py`
- **治理项:** 针对 `_apply_vibrance` 与 `_apply_hue` 中的 `hsv.split()` 通道 Image 对象，显式添加了 `ch.close()` 释放，避免大量调色操作时 PIL 图像句柄在内存中堆积。

#### (2) 安全与沙箱机制加强
- **机制确认:**
  - `engine_loader.py` 白名单限制 `ALLOWED_ENGINES` 拒绝非法模块导入。
  - `assert_svg_render_safe` 校验 XML 构建，全面阻断 XXE 实体注入与外部链接引用。
  - `window.py` 文件拖拽桥接统一使用 `json.dumps(payload, ensure_ascii=True)`，彻底杜绝 JS 注入漏洞。
  - `settings_store.py` 采用 `mkstemp` + `os.replace` 原子替换写入设置文件，防止断电导致配置损坏。
